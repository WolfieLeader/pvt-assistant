# Retro: Setup (Phases 1–6)

**Dates**: 2026-03-01 → 2026-03-03

## Architecture & Code Organization

- Feature-module pattern (`src/features/<name>/`) — isolate crypto, store, components, hooks behind a barrel; only export public API (hooks, types)
- Co-locate adapters with their dependency — `mmkvStorage` belongs in `utils/mmkv.ts` alongside the MMKV instance, not in `stores/`
- Side-effects belong in providers, not bare hooks — `useTheme` became read-only after extracting `Uniwind.setTheme()` into ThemeProvider
- When a shared dir (`components/ui/`) contains only flat primitives with a barrel, flatten it — nesting adds import depth without organizational value
- Don't create `src/services/` for feature-specific logic — crypto belongs in the feature module
- Implementation order matters — build leaf components first (Text), then dependents, to avoid circular deps
- Compose hook results into a single `useAppReady` gate — keeps app entry clean

## Styling & Design System

- Dual token layer (CSS vars + JS consts) separates NativeWind styling from imperative APIs — every CSS token group needs a corresponding JS const file
- Check `global.css` `@theme` for existing Tailwind utilities before using arbitrary `[value]` syntax
- Hoist inline style objects to module-level constants for hot-path components (Text, Screen)
- Document color generation tools/URLs in design-system.md so future palette changes follow the same process

## Reanimated & Animations

- Reanimated hooks have native-side cost even when unused — gate behind conditional rendering (inner component) for dual-mode components
- `withRepeat(..., -1, ...)` needs `cancelAnimation` cleanup on unmount — infinite animations leak without it
- Shared `usePressAnimation` hook eliminated ~45 lines of duplicated Reanimated logic across 3 components

## React Native Platform

- Haptics are platform-divergent: iOS = physics-based (light/medium/heavy), Android = semantic (confirm/reject/toggle) — design haptic maps with both models
- RN can't resolve font weights from a single family name — always use explicit per-weight font family names
- Prefer expo-crypto over Web Crypto API in RN — no polyfill needed, works synchronously for hashing
- `crypto.subtle` requires a polyfill in RN; don't reach for it when expo-crypto covers the need

## Drizzle & SQLite

- `$defaultFn` is cleanest for JS-side defaults — avoids SQL expression gymnastics
- SQLite FK enforcement is opt-in per connection (`PRAGMA foreign_keys = ON`) — easy to miss
- Enum ints over strings with const maps in `enums.ts` — smaller storage, type-safe, labels decoupled
- Amount in cents (int) not floats — avoids floating point math
- nanoid for user-facing IDs, autoIncrement int for internal — readable URLs, fast joins
- Seed idempotency via `count() === 0` — simple, prevents duplicate data on re-opens

## Type Safety

- Derive types from library APIs (`Parameters<typeof Fn>[0]`) rather than duplicating — stays in sync automatically
- Zod shorthand utils (`zStr`, `zText`, `zCents`) — DRY form validation without repeating constraints

## Component Patterns

- When wrapping a component, either use its API properly or bypass it with the raw primitive — don't set defaults you'll immediately override
- App-provided recovery questions > user-written questions — curated list ensures quality
- PIN: 6-digit minimum; escalating lockout (30s → 1min → 5min) makes brute-force impractical even without long PINs

## Tooling & Workflow

- Always use `just verify` not bare `tsc --noEmit` — catches formatting and lint issues typecheck alone misses
- MMKV v4 API: method is `remove(key)`, not `delete(key)`
- Cross-check plan completeness before calling a phase done
- Capture planning decisions (color tools, API research) in docs — lost context costs rework
